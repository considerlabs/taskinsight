## 5. 보안 요구사항

| 항목 | 요구사항 |
|---|---|
| 전송 암호화 | HTTPS 필수 (사내망이어도 TLS 사용) |
| 비밀번호 | bcrypt 해시 (rounds=12) |
| JWT 만료 | Access Token 15분, Refresh Token 30일 |
| 계정 잠금 | 5회 실패 → 15분 잠금 |
| SQL Injection | Parameterized query 필수 (f-string 금지) |
| XSS | 프론트엔드 출력 시 이스케이프 (React 기본 처리) |
| 파일 업로드 | MIME 타입 + 확장자 이중 검증 |
| 실행 파일 차단 | .exe .bat .sh 등 서버 사이드에서 차단 |
| API Rate Limit | 로그인 실패: 분당 10회 제한. 일반 API: 분당 300회 제한. |
| 인증 헤더 | Authorization: Bearer. 쿼리 파라미터 토큰 허용 안 함. |
| CORS | 사내 서버 도메인만 허용 (APP_BASE_URL 기준) |

## 6. 백업 정책

| 항목 | 정책 |
|---|---|
| DB 백업 주기 | 매일 새벽 3시 자동 전체 백업 |
| 백업 보관 기간 | 최근 30일 |
| 백업 저장 위치 | 서버 내 별도 볼륨 + 외부 NAS 또는 클라우드 스토리지 (선택) |
| 백업 검증 | 월 1회 복구 테스트 실시 |
| 파일 업로드 백업 | Docker volume 포함 |
| 백업 알림 | 실패 시 Teams + 이메일 알림 |

```bash
# DB 백업 스크립트 (cron: 0 3 * * *)
docker-compose exec -T db pg_dump -U taskinsight taskinsight | \
  gzip > /backup/taskinsight_$(date +%Y%m%d).sql.gz

# 30일 이상 된 백업 삭제
find /backup -name "taskinsight_*.sql.gz" -mtime +30 -delete
```

## 9. 유지보수성
